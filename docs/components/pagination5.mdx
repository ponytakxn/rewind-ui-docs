### Settings

The internal configuration of the component works as follows:

The slug is what comes before the page number. If the slug is empty then the pagination will look like this `/pageNumber/`. To have a custom slug you must enter it ending with `/`, for example if we want to put a slug for a page `products` the slug should look like this: `products/`, which would result in `/products/pageNumber`.

On the other hand, handlePagination is the function that will handle the change in pagination. Internally the pagination buttons have a function called as follows `handlePagination(page+1)`.
